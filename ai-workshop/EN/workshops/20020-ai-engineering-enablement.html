20020 · AI-assisted Engineering 06/2026

Practical workshop for technical teams in mid-sized companies.

Using AI as a guided engineering tool: with precise prompts, architecture context, review and human responsibility.

Enablement

Core idea

This workshop is not AI marketing. It is not about buzzwords, frameworks or prompt hacks. It focuses on real engineering problems: how can AI help with actual technical work?

  • Reality instead of hype: capabilities, limits and hallucinations
  • Increase productivity in everyday engineering work
  • Use prompt engineering as a structured engineering skill
  • Structure requirements, challenge assumptions and detect gaps
  • Document architectures, interfaces and decisions
  • Prepare security reviews and support security-by-design
  • Evaluate cloud services, local models and RAG systems

Audience

Engineering managers, architects, embedded developers, security leads, project managers and senior engineers.

Format

4 hours, remote or on-site, interactive, around 6 to 20 participants.

Which problem does this workshop solve?

Engineering teams are asked to use AI, but often lack clear rules, prompting practices, review habits and architecture context. This workshop shows how AI can support real engineering work without reducing quality, responsibility or security.

Who is it for?

For development leads, system architects, software architects, embedded developers, security leads, project managers, senior engineers and technical decision makers in mid-sized engineering organizations.

What do participants take away?

Prompting & LLM basics

  • Understand how prompts work
  • Write precise and reviewable prompts
  • Apply basic rules for LLMs in engineering
  • Assess capabilities and limits realistically
  • Treat outputs as proposals, not as truth

Planning & architecture guidance

  • Plan work in fine-grained steps
  • Use AI to support and challenge plans
  • Structure requirements
  • Provide architecture context deliberately
  • Keep structure and architecture as the foundation
  • Document decisions in a traceable way

Coding rules & review

  • Define rules for AI-assisted coding
  • Never accept AI-generated code without review
  • Keep responsibility with the developer
  • Review code, architecture and security deliberately
  • Use the tool, but stay skeptical of the output

Cost, data & tools

  • Account for long sessions, commands and automated analysis
  • Address privacy and IP protection
  • Do not treat unprepared data as a reliable foundation
  • Compare cloud services, local models and RAG systems
  • Choose tools based on workflow and risk

Why is ChatGPT alone not enough?

AI does not replace engineers, architects or critical thinking. It becomes useful only when people provide structure, context, review and technical judgment. Vibe coding without guidance produces arbitrary code; good AI-assisted engineering needs rules, architecture and responsibility.

How does the workshop run?

The workshop combines short technical explanations with practical examples: requirements analysis, documentation, architecture questions, research, code review, security review, local and cloud tools, cost considerations and privacy boundaries. The examples are based on real engineering work, not prepared marketing demos.

Which prerequisites are needed?

Participants do not need to be AI specialists. Useful preparation includes current engineering tasks, documentation examples, review questions, requirements, architecture topics or concerns around source code, privacy and tooling.

What result is available at the end?

Participants understand where AI can help in their daily work, where it creates risk, how outputs should be reviewed and which working rules are needed for productive use in engineering teams.

Which workshop fits afterwards?

If security, local models or RAG become relevant, continue with Secure AI Engineering. If organizational rules are needed, continue with AI Governance & Responsible AI.
