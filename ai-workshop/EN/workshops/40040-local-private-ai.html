40040 · Secure AI Engineering

Integrating AI systems securely into engineering processes.

From local Large Language Models (LLMs) and Retrieval-Augmented Generation (RAG) to privacy, source code protection, security by design and architecture.

Security

Topics

  • Evaluate local LLMs and cloud services securely
  • Plan RAG systems on company knowledge
  • Protect privacy, confidentiality and source code
  • Anchor security by design in AI-assisted workflows
  • Prepare architecture and integration into engineering processes

Focus

Technical feasibility combined with security and privacy boundaries.

Outcome

A realistic target picture for local AI in your environment.

Which problem does this workshop solve?

Engineering organizations need AI support without uncontrolled data flows, unclear model choices or accidental source code exposure. This workshop clarifies how AI can be integrated securely into technical work.

Who is it for?

For system architects, software architects, security leads, CTOs and engineering managers who evaluate local LLMs, RAG systems, cloud tools or secure AI-assisted development workflows.

What do participants take away?

  • A clear comparison of cloud services, local models and RAG systems
  • An understanding of privacy, confidentiality and IP risks
  • Architecture considerations for secure AI integration
  • Practical guardrails for source code and company knowledge

Why is ChatGPT alone not enough?

Secure AI engineering is not only a user interface question. It requires decisions about model operation, retrieval architecture, data boundaries, access control, logging, review and security-by-design.

How does the workshop run?

We review realistic operating models, local LLM options, RAG architectures, source code handling, privacy boundaries and integration into existing engineering processes. The result is a technical target picture, not a tool recommendation in isolation.

Which prerequisites are needed?

Helpful inputs are current AI tools, security requirements, privacy constraints, source code policies, documentation sources and existing infrastructure constraints. A finished AI architecture is not required.

What result is available at the end?

The team receives a practical view of feasible architectures, risk boundaries, required decisions and the next steps for a secure AI setup.

Which workshop fits afterwards?

For team enablement, continue with AI-Assisted Engineering. For rules and accountability, continue with AI Governance & Responsible AI.
