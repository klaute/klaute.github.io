30030 · Secure AI Engineering

Integrating AI systems securely into engineering processes.

From local Large Language Models (LLMs) and Retrieval-Augmented Generation (RAG) to data flows, source code protection, technical system boundaries, operating options and architecture decisions.

Security

Topics

  • Evaluate local LLMs and cloud services securely
  • Assess Retrieval-Augmented Generation (RAG) as an architecture element
  • Describe data flows, system boundaries and access protection
  • Consider source code protection, confidentiality and protection of intellectual property
  • Classify technical guardrails, review points and logging
  • Compare operating and integration options for engineering landscapes

Focus

Technical feasibility, data flows, access protection and security boundaries.

Outcome

A technical target picture with open decisions, risks and next integration steps.

Which problem does this workshop solve?

Engineering teams want to use AI-assisted engineering, but need to protect source code, internal knowledge, confidential data and intellectual property. This workshop clarifies how a technical solution can be built: with system boundaries, data flows, access protection, operating options and architecture decisions.

Who is it for?

For architects, security leads, engineering leads and platform or infrastructure teams that need to evaluate local LLMs, RAG or cloud services and prepare technical integration steps.

What do participants take away?

  • A technical classification of local models, RAG and cloud services
  • A view of data flows, access protection and source code protection
  • Operating options and integration paths
  • Technical review points, logging and security boundaries

Why is ChatGPT alone not enough?

ChatGPT does not answer which data flows where, which protection boundaries apply, how knowledge can be indexed, which access controls are required or how a solution should be integrated into existing engineering landscapes.

How does the workshop run?

We review architecture variants, operating models, local models, RAG, data flows, access protection, source code protection and integration points in the engineering environment. This includes technical guardrails, logging, technical approval points and handling of public, internal and confidential knowledge sources.

Which prerequisites are needed?

Helpful inputs are existing infrastructure, data sources, source code use, security requirements, operating constraints and current AI tools. Legal or regulatory requirements must be clarified by the customer's responsible functions.

What result is available at the end?

The team receives a traceable technical target picture with architecture and operating options, data and security boundaries, open decisions, risks and next integration steps.

How is it different from the other workshops?

AI Readiness for Engineering clarifies starting point, use cases and roadmap. AI-Assisted Engineering defines working practices, review rules and responsibility within the team. This workshop focuses on technical architecture, operating models, data flows, access protection and security boundaries. It is not legal advice, GDPR interpretation, EU AI Act consulting or compliance certification.
