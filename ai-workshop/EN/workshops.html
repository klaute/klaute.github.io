Workshop Portfolio for AI-Assisted Engineering 07/2026

Workshops for AI-Assisted Engineering from an architecture perspective.

The workshops are not AI coaching and not tool training. They help technical organizations implement AI-assisted engineering through a systemic lens: with local Large Language Models (LLMs), Retrieval-Augmented Generation (RAG), knowledge structure, security by design and clear guardrails for engineering teams.

What this is about

Many companies start with copilots or isolated model experiments. AI-assisted engineering becomes sustainable only when architecture context, data flows, confidential source code, knowledge modeling, reviews and responsibilities are treated as connected infrastructure.

Orientation

  1. Which complexity should be reduced?
  2. Which data and knowledge assets may be used?
  3. Which architecture protects source code and IP?
  4. Which guardrails enable teams in daily work?

Which entry point fits?

If it is still unclear which role AI should play in the organization, start with strategy and readiness. If confidential source code, local models or RAG are central, start with secure AI engineering. If teams already use AI, AI-assisted engineering is the entry point into rules, review and architecture context.
